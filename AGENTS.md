# Codex项目规则

## 项目目标

这是“骑不快的ZZ（大熊说车）”的性格选车测试。账号定位是：懂物理、懂机械、看透行业水深的现实判断型机车知识账号，帮助普通摩友少摔车、少亏钱、少踩坑。

## 不允许改坏的部分

1. 保留一题一屏访谈，不改回一页长表单。
2. 保留ZZ主持人实时判断。
3. 结果必须讲“代价、风险、适合与不适合”，不能只堆参数。
4. 不把不完整车型数据包装成精确推荐。
5. 无手续水车、报废车不进入日常上路推荐。
6. 总访谈不超过12轮，新增问题应优先合并到现有复合题。
7. 不上传身高、体重和答案；默认只保存在浏览器。
8. UI保持深蓝、夕阳橙、复古机车海报感，不改成通用科技蓝或廉价渐变风。

## 修改优先级

1. 数据准确性与字段完整度。
2. 推荐解释和硬过滤。
3. 手机端完成率。
4. 结果分享与账号导流。
5. 视觉细节。

## 数据规则

- `data/base-vehicles.js`：基础示例库。
- `src/vehicles.generated.js`：公开页采集生成库。
- 所有车型必须经过 `normalizeVehicle()`。
- 新字段先更新 `src/engine.js` 的归一化，再更新UI。
- 价格、座高、整备质量、年款、在售状态必须允许为空；为空时显示“待校准”。
- 不得编造空缺数据。

## 完成标准

每次修改后运行：

```bash
npm run check
npm run dev
```

至少完成：首页、12轮访谈、结果页、车型弹窗、海报弹窗的手机和桌面测试。
