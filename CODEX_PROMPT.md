# 复制给Codex的执行任务

请先完整阅读 `AGENTS.md`、`README.md` 和现有代码，然后执行：

1. 运行 `npm run check`，修复所有语法、数据结构和测试问题。
2. 运行 `npm run dev`，用浏览器完整点一遍12轮访谈。
3. 分别检查桌面宽度1440px和手机宽度390px。
4. 检查以下关键路径：
   - 通勤新手、低预算、低持有成本。
   - 188cm以上、高个、撑场面。
   - 165cm以下、摩旅、喜欢ADV。
   - 完全新手、极端动力。
   - 收藏模式、低维护意愿。
   - 露天停车、高价值车型。
5. 修复交互问题，但不要改回长表单。
6. 优化手机端：选项点击面积、底部导航、进度、结果卡片、弹窗和海报。
7. 检查推荐理由是否能解释“为什么推荐”和“为什么不推荐”。
8. 运行摩托范采集脚本前，先用 `MAX_DETAILS=30` 试跑；不要高并发，不要绕验证码。
9. 采集后运行归一化，再执行 `npm run check:data`。
10. 数据为空时必须显示待校准，禁止自动编造座高、车重、年款和在售状态。
11. 最后执行 `npm run build`，确认 `dist/` 可以直接部署。

交付时给出：

- 修改文件清单。
- 已修复问题。
- 尚未校准的数据字段。
- 本地运行地址。
- GitHub Pages部署步骤。
