# 骑不快的ZZ｜选车与装备建议 V6.5.2

这是一个纯前端的摩托车与骑行装备选择工具。用户每次只选择一个问题：完成 12 道选车题，或单独测试头盔、手套、骑行服/护具、骑行靴、尾箱/箱包、辅助灯、头盔耳机和防盗方案。

## V6.5.1 重点

- 保留 33 条车型方向和既有的国内在售/停售/水车安全门禁。
- 装备资料库扩充到 583 条，覆盖头盔、手套、骑行服/骑行裤/护具、骑行靴、箱包、辅助灯、头盔耳机和防盗八类。
- 头盔问卷新增盔型、头围和头型；骑行服问卷新增网眼、拉力服、分体/连体皮衣、护甲衬衣、骑行裤、体型/版型；手套和骑行靴也补充手型、脚型与靴筒适配。
- 页面将“最接近你的需求”与“品牌 → 产品线 → 型号资料库”分开显示。10 条优先浏览候选不再被写成整个目录，可按类型、用途、品牌和资料状态筛选。
- 默认匹配只使用中文官方/旗舰店的具体型号资料；京东榜单、价格页和品牌类目页只作为有日期的公开出现快照或目录来源，不冒充实时库存。
- 找不到中文具体型号资料时，页面会改成“选型参考（国内购买待核验）”；品牌—产品线—型号资料库仍完整保留，便于继续核对。
- 头盔与骑行服选择会强制匹配盔型/服装结构，不会把拉力盔、赛道皮衣等跨类型混入结果。
- 公开平台快照必须显示观察日期和来源。没有快照的条目只标为目录候选，不冒充销量排名。
- 产品阶梯会先检查核心筛选条件；没有符合条件的替代项时，允许只显示一至两项。
- 高价值露天防盗按互补防护层处理，AirTag 不会成为主方案。
- 国内渠道、当前价格或头盔 GB 811-2022 具体型号证据不充分时，页面会明确提示再次核验。
- 首页、问题、结果页、弹窗和按钮文案已统一为自然、可读的表达。

## 数据边界

583 条是可维护的品牌、产品线和型号资料范围，不等于中国市场的全部 SKU，也不代表每条都已经核验为国内现货。页面会把中文官方具体型号资料、有日期的公开平台快照、选型参考和仅供浏览条目分开标注；价格、库存、版本、认证和销售渠道会变化，购买前必须以品牌、监管平台和正规销售渠道的最新信息为准。

## 本地运行

```powershell
npm.cmd run check
npm.cmd run build
npm.cmd run dev
```

打开 `http://127.0.0.1:4173`。

## 部署

推送 `main` 后，[GitHub Actions](.github/workflows/pages.yml) 会依次执行全部检查、严格白名单构建，并把 `dist/` 部署到 GitHub Pages。

构建脚本只发布页面必需文件，不会把原始爬虫数据、审核账本或内部资料复制到 `dist/`。


## V6.5.2 补充

本版新增“近似替代”展示逻辑：当没有完全匹配的装备候选时，页面会明确说明，并给出保留核心使用场景的近似方向，避免结果页显得不完整。
