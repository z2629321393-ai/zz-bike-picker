# V6.5.2 更新说明

本次更新重点解决“没有完全匹配时页面看起来不完整”的问题。

## 已调整

1. **近似替代逻辑**
   - 当头盔、手套、骑行服、骑行靴、箱包、蓝牙耳机没有完全匹配的可核验候选时，页面会明确说明“没有完全合适的，先看近似方向”。
   - 不再直接留空，也不再给人“像是漏推荐”的感觉。

2. **头盔类型回退**
   - 半盔缺少合适型号时，优先回退到 **3/4盔**。
   - 3/4盔缺少合适型号时，回退到 **复古全盔 / 街道全盔**。
   - 拉力/ADV盔缺少合适型号时，回退到 **GT/街道全盔 / 运动全盔**。
   - 越野盔缺少合适型号时，回退到 **拉力盔 / 全盔**。

3. **画面提示**
   - 近似替代模式下，示意图上的提示改成“没有完全合适的，先看近似方向”。
   - 结果页标题会明确显示“当前没有完全适合的XX，先看近似方向”。

4. **其他类目统一**
   - 手套、骑行服、骑行靴、箱包、耳机也采用类似策略。
   - 近似替代只在守住核心使用场景的前提下发生，不会跨到明显不适合的用途。

## 验证结果

- `npm run check`：通过
- `npm run build`：通过
